Описание

Определение требований для перехода на следующую компетенцию является важным этапом в процессе развития. Это позволяет установить ясные и конкретные цели, которые необходимо достичь, чтобы продвинуться дальше.

Цели

1. Оценка текущего уровня знаний и навыков:

  • Определение требований позволяет оценить, какие знания и навыки уже есть у человека, и определить области, в которых необходимо дальнейшее развитие.
  • Это помогает понять, какие компетенции нужно усовершенствовать или приобрести, чтобы перейти на следующий уровень.

2. Установление реалистичных целей:

  • Определение требований помогает установить реалистичные и достижимые цели для перехода на следующую компетенцию.

3. Планирование обучения и развития:

  • Зная требования для перехода на следующую компетенцию, можно разработать Индивидуальный план развития, который будет наиболее эффективным и подходящим для конкретного человека.
  • Это может включать в себя изучение новых материалов, посещение курсов, мастер-классов, общение с экспертами и т.д.

4. Мотивация и самодисциплина:

  • Определение требований может служить мотивацией для дальнейшего развития и обучения.
  • Человек видит конкретные цели, к которым он стремится, и это помогает поддерживать самодисциплину и настойчивость в достижении этих целей.

5. Оценка прогресса и корректировка плана:

  • Регулярная оценка прогресса и сравнение его с установленными требованиями позволяет увидеть, насколько успешно человек продвигается к следующей компетенции.
  • В случае необходимости можно внести коррективы в план обучения и развития, чтобы ускорить процесс или изменить подход.

Составление матрицы компетенции

1. Создайте папку на drive с доступом для сотрудника, для которого составляется матрица компетенций. В данную папку сотрудник сможет загружать примеры документаций, СТ, инструкций, отзывов.

2. Создайте новый документ Excel. Скопируйте в документ требуемые пункты по соответствующему уровню компетентности, на который планируется повысить сотрудника. Перечень компетенций установлено в файле «Матрица компетенций РСБУиФУ»

3. Вариант шапки документа:

  • Компетенция – берется из файла «Матрица компетенций РСБУиФУ», если для соответствующего уровня компетентности установлено значение 1.
  • Требования – Фиксированное значение 1 (для определения суммарного количества компетенций и расчета требуемого количества компетенций)
  • На текущий момент – Устанавливается значение:
    • 1, если сотрудник обладает данной компетенцией
    • 0, если сотрудник не обладает данной компетенцией
  • Требуется – Устанавливается значение:
    • 1, если данная компетенция является обязательной для перехода на следующий уровень
    • 0, если данная компетенция является необязательной для перехода на следующий уровень
  • Комментарий – Ссылки, дополнительное описание, рекомендации.
  • Требуется от РГ – Описание действия руководителя группы по зачету данной компетенции
  • Примеры – Описание примеров для зачета компетенции (примеры задач для обсуждения, пройденные курсы)

4. Определите количество требуемых компетенций для перехода на следующий уровень: 80% от максимума

5. Поместите Матрицу компетенций в отдельную папку. В дальнейшем в этой папке можно будет размещать дополнительные файлы по матрице: Примеры СТ, инструкций, отзывов. Ссылку на папку необходимо указать в файле руководителя отдела https://drive.rarus.ru/f/25875439. Для этого необходимо нажать на «Скопировать общедоступную ссылку»:

Вариант доступа к папке для сотрудника: добавить его как получателя общего доступа, дать права для редактирования и добавления. (Договорились, что Руководителю отдела не будем давать доступ таким образом, чтобы не увеличивать количество папок у нее на drive)

6. Согласуйте Матрицу компетенций с руководителем отдела.

7. Обсудите Матрицу компетенций с сотрудником (пригласите на встречу руководителя отдела).

8. Актуализируйте матрицу компетенций, отслеживайте прогресс.

9. При достижении необходимого количества компетенций обсудите Матрицу компетенций с руководителем отдела о возможности повышения уровня компетентности сотрудника.
